# Руководство пользователя

## 1. Управление через CLI (`lastex.py`)

Управление проектом осуществляется через скрипт `lastex.py`, который транслирует команды в фоновый Docker-контейнер.

*   `python lastex.py build <путь_к_директории>` — компиляция LaTeX-документа. **Внимание:** Python-скрипты с графиками (т.е. те, что не включены в `.tex` документ импортом) автоматически не выполняются.
*   `python lastex.py build <путь_к_файлу.py>` — изолированный запуск конкретного скрипта с автоматическим перемещением результатов в директорию `figs/`.
*   `python lastex.py clean <путь>` — удаление директорий `out/` и `figs/` (очистка кэша).
*   `python lastex.py stop` — принудительная остановка и удаление фонового контейнера.
*   `python lastex.py status` — проверка состояния контейнера.

## 2. Интеграция с VS Code

Система поддерживает контекстно-зависимую сборку через горячие клавиши (требуется настройка `tasks.json` и `keybindings.json`).
*   **Smart Build** (`Ctrl + Alt + B`):
    *   При активном `.py` файле: перестраивает только текущий график.
    *   При активном `.tex` файле: инициирует полную сборку всего отчета.

## 3. Структура отчета и конфигурация титульного листа

Точкой входа для каждого отчета является файл `_report.tex`. В нем задаются метаданные для титульного листа. Использование `\documentclass` обернуто в условный оператор для совместимости с прекомпилированным форматом.

См. шаблон в [`template/_report.tex`](../template/_report.tex).

## 4. Оформление документа (Макросы и Сниппеты)

### 4.1. Секции и ссылки
*   Центрированный заголовок без нумерации (для введения/заключения): `\centeredsection{НАЗВАНИЕ}`.
*   Перекрестные ссылки: строго через команду `\cref{метка}`. Пакет `cleveref` автоматически подставляет слова «рис.», «табл.», «формула».

### 4.2. Таблицы (`tabularray`)
В проекте используется пакет `tabularray`.

Пример создания таблицы со сниппетом `tblr`:
```latex
\begin{table}[H]
    \centering
    \caption{Результаты измерений}
    \label{tab:results}
    \begin{tblr}{
        colspec = {| Q[c, m] | Q[c, m] | X[j, h] | },
        hlines = 1, vlines = 1,
        row{1} = {font=\bfseries},
    }
        $U$, В & $I$, А & $R$, Ом \\
        $10$ & $0.1$ & $100$ \\
    \end{tblr}
\end{table}
```

## 5. Вычислительный конвейер: Python $\rightarrow$ $\LaTeX$

⚠️ **Внимание:** Использование Lua (`\val`, `\luablock`, `\luarow`) объявлено устаревшим. Новые проекты должны использовать Python. Директория `code/` запрещена из-за конфликта с системными библиотеками Python, используйте `scripts/`.

Для автоматизации расчетов используется фоновый процесс Python (`pyluatex`).
Рекомендуемый процесс работы:
1.  **`scripts/protocol.py`**: инициализация массивов (списки, numpy-массивы) с сырыми данными.
2.  **`scripts/processing.py`**: математическая обработка массивов.
3.  **`_report.tex`**: инициализация окружения через `\begin{python} from scripts.processing import * \end{python}`.
4.  **`protocol.tex` / `processing.tex`**: вывод данных через макросы `\pv` (Python Value) и `\pt` (Python Table).

### Форматирование значений (Синтаксис `[a].[b][c]`)
Макросы принимают строку формата, состоящую из трех опциональных частей (`\pv*[a.bc]{somevalue}`):
*   `a`: Масштабирование. Делит число на $10^a$. (`0.456` -> `-3.` -> `456`)
*   `b`: Округление. Количество знаков после запятой (если $<0$ — округление до десятков/сотен).
*   `c`: Экспонента. Символ `e` выводит число в виде $M \cdot 10^{exp}$.

*Примеры:*
*   `[.2]` — 2 знака после запятой.
*   `[3.1]` — разделить на $10^3$ (перевод в кило-единицы), оставить 1 знак.
*   `[.2e]` — экспоненциальная форма, 2 знака в мантиссе.

### Пример использования `\pt` в таблице
Макрос `\pt` разворачивает массивы Python в строки таблицы. Нумерация строк в LaTeX начинается с 1, конвертация в Python-индексы (с 0) происходит автоматически.

\begin{table}[H]
    \centering
    \begin{tblr}[expand=\py]{
        colspec = { X[c,m] X[c,m] X[c,m] },
        hlines, vlines,
    }
        $U$, В & $I$, мА & $R$, кОм \\
        % Вывод первых двух элементов. I масштабируется в мА (a=-3), R в кОм (a=3)
        \pt*[.1, -3.2, 3.2][1][2]{U_arr, I_arr, R_arr} \\
        \SetCell[c=3]{c} \textit{Смена диапазона} \\
        % Вывод с 3-го элемента до конца
        \pt*[.1, -3.2, 3.2][3][-1]{U_arr, I_arr, R_arr} \\
    \end{tblr}
\end{table}

### Сниппеты VS Code
*   `pe` — блок `\begin{python}...\end{python}`
*   `py` — инлайн вызов `\py{...}`
*   `pv` — макрос одиночного значения `\pv*[.]{var}`
*   `pt` — макрос таблицы `\expanded{\pt*[.]{arrays}}`

См. [`core/templates/etulab/python/python.sty`](../core/templates/etulab/python/python.sty) для более детальной документации.

## 6. Подсистема графики (Python $\rightarrow$ PGF)

⚠️ **Архитектурное правило:** Логика вычислений (`scripts/processing.py`) и скрипты генерации графиков (`scripts/plot_*.py`) **должны быть разделены**.

Графики не перестраиваются автоматически при компиляции отчета для экономии времени. Для построения графиков используется Matplotlib с бекендом PGF.
1.  Скрипт генерации графика (например, `plot_afc.py`) должен импортировать готовые данные из `processing.py`.
2.  Запуск скрипта производится **вручную**: через CLI (`python lastex.py build путь_к_файлу.py`) или по `Ctrl + Alt + B` в VS Code (при открытом `.py` файле).
3.  Сгенерированные `.pgf` файлы автоматически перемещаются скриптом `lastex.py` в директорию `figs/` относительно корня текущего отчета.
4.  Глобальные настройки (толщина линий, сетка) берутся из [`core/matplotlibrc`](../core/matplotlibrc).

Вставка графика в документ:
```latex
\begin{figure}[H]
    \centering
    \input{figs/plot.pgf}
    \caption{Вольтамперная характеристика}
    \label{fig:vac}
\end{figure}
```

См. пример использования в [`reports/sem_4/em/lab-No2__3/code/`](../reports/sem_4/em/lab-No2__3/code/)

## 7. Структура отчета
Промпт для ИИ ([`api_respose.md`](./AI/api_response.md)) оптимизирован для следующей структуры:
```
some-lab/
├── code/
│   ├── plot1.py
│   ├── plot2.py
│   ├── ...
│   ├── plotn.py
│   ├── processing.lua
│   └── protocol.lua
├── figs/
│   ├── plot1.pgf
│   ├── plot2.pgf
│   ├── ...
│   ├── plotn.pgf
│   ├── some.png
│   └── some.pdf
├── _report.tex
├── main.tex
├── processing.tex
└── protocol.tex
```

При этом все импорты указываются относительно директории `some-lab/` (см. [`api_respose.md`](./AI/api_response.md) п.4, п.5).
