# Архитектура проекта и конвейер сборки

Система LasTeX спроектирована для минимизации времени компиляции тяжелых документов и обеспечения идентичности среды на любой ОС.

## 1. Изоляция среды (Docker)

Базовый образ: [`registry.gitlab.com/islandoftex/images/texlive:TL2025-2025-11-02-small`](https://gitlab.com/islandoftex/images/texlive).
Контейнер `latex-daemon` запускается в фоновом режиме (`sleep infinity`). Это исключает накладные расходы на поднятие контейнера при каждой сборке. Локальная директория проекта монтируется в `/workdir`.

Установленные компоненты:
*   LuaLaTeX, Biber, Latexmk.
*   Пакеты: `tabularray`, `minted`, `circuitikz`, `pgfplots` и др.
*   Python $3$ с библиотеками `numpy`, `matplotlib`, `scipy`, `lupa`.

## 2. Интеллектуальный конвейер сборки (`lastex.py`)

Процесс сборки инициируется на хост-машине, но все операции выполняются внутри контейнера.

### 2.1. Генерация нативного формата
Для ускорения компиляции класс `etulab.cls` и все подключаемые пакеты компилируются в бинарный дамп `etulab_fmt.fmt`. 
*   При запуске `lastex.py` проверяет дату изменения `etulab.cls` и существующего `.fmt`.
*   Если локальный класс новее, инициируется пересборка формата командой `lualatex -ini`.
*   При сборке документа формат загружается первой строкой `%&etulab_fmt`, минуя этап парсинга преамбулы.

### 2.2. Параллельная обработка Python-скриптов
Перед запуском LaTeX-компилятора скрипт ищет все файлы `.py` глубиной до $2$ уровней. Выполнение происходит параллельно через утилиту `xargs -P$(nproc)`.
Сгенерированные `.pgf` файлы автоматически маршрутизируются в директорию `figs/`.

### 2.3. Компиляция документа
Сборка управляется утилитой `latexmk` с флагом `-pdflua`. Вывод фильтруется через `texfot`, оставляя в консоли только критические ошибки (Overfull/Underfull, Missing characters, Errors). Имя выходного файла генерируется динамически функцией `get_jobname`, сохраняя иерархию директорий (например, `sem_4__physics__lab_01.pdf`).

## 3. Синхронизация конфигураций
Файл [`core/matplotlibrc`](../core/matplotlibrc) автоматически копируется в `/etc/matplotlibrc` внутри контейнера при каждом вызове `build`. Это гарантирует применение академических стандартов графики ко всем Python-скриптам без необходимости прописывать `plt.rcParams` в каждом файле.
