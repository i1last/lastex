**Язык вывода: строго русский.**

## 1. Технический интерфейс (API)
Используй макросы пакета `lua.sty` для связи с расчетным модулем:
- `\val[формат]{путь}` — одиночное значение (основной и предпочтительный способ). Умеет полноценно обрабатывать lua.
- `\luarow[формат]{путь}[старт][конец]` — генерация строки (разделитель `&`).
- `\luacol[формат]{путь}[старт][конец]` — генерация столбца (разделитель `\\`). Принимает список путей через запятую.
- `\luablock[форматы]{пути}[старт][конец]` — генерация матрицы. Принимает списки форматов и путей через запятую.

**Важное правило:**  
Всегда используй `\val` вместо `\directlua{tex.print(...)}` или `\luaexec`, если возможно.
`\directlua` разрешается только в исключительных случаях (сложные условные конструкции или динамическое формирование текста).

**Форматы:**
- `[n]` — количество знаков после запятой (например, `[2]`).
- `[exp]` — экспоненциальная форма.
- Для `\luablock` форматы соотносятся с путями по порядку.

**Обязательное правило для таблиц:**  
Макросы `\luarow`, `\luacol`, `\luablock` всегда оборачиваются в `\expanded{...}`.

## 2. Архитектура файлов и логика данных
- `protocol.tex`: Таблицы для «сырых» данных. Минимум теории.
- `main.tex`: Цели, задачи, сжатая теория. Подробнейшая инструкция по снятию данных (Что сделать → Как → Нюансы).
- `processing.tex`: Последовательное выполнение заданий. В конце файла **ОБЯЗАТЕЛЬНО** оставить пустой раздел `\section*{Выводы}`.
- `processing.lua`: Логика расчетов. Импортирует `protocol.lua`.

**Требования к Lua-переменным:**
- Используй максимально короткие, но семантически понятные имена переменных.
- Данные из протокола хранятся в глобальной таблице `p` (protocol).
- Все результаты обработки хранятся в глобальной таблице `r` (results).
- **Все таблицы (`p` и `r`) обязательно должны быть глобальными** — это критично для корректной работы `\val`, `\luarow` и `\luablock`.

**Оформление заданий в `processing.tex`:**
    ```latex
    \subsection*{ \boxed{\text{ Задание N. }} }
    \begin{quote}
        \textit{<Текст задания из методички>}
    \end{quote}
    ```

## 3. Правила верстки, таблиц и подписей
- **Подписи к элементам:**
  - Таблицы и листинги (`table`, `longtblr`, `listing`) — `\caption{}` **всегда сверху**.
  - Рисунки (`figure`) — `\caption{}` **всегда снизу**.
- **Расширение таблиц:** Для `tblr` и `longtblr` по умолчанию `expand=\expanded`.
- **Объединение ячеек:** Только `\SetCell[r=rows, c=cols]{align}`.
- **Списки:** Только окружения `itemize` или `enumerate`. Запрет на Markdown-нумерацию.
- **Выделение:** Только `\textit{...}`. Полный запрет на `\textbf{...}`.
- **Ссылки:** Только `\cref{label}`.
- **Математика:** Инлайн `$ ... $`, выносные `equation`/`align`. Запрет на `$$ ... $$`. Числа через точку (`3.14`).

## 4. Требования к обработке в `processing.tex`
**Главное правило автоматического пересчёта:**
- Все числа, коэффициенты, результаты и итоговые значения в `processing.tex` должны браться исключительно из Lua (`\val`, `\luarow`, `\luablock` и т.д.).
- Запрещено вписывать числа вручную (даже примерные или округлённые).
- Это гарантирует, что при изменении данных в `protocol.lua` вся лабораторная работа пересчитается автоматически и полностью.

**При этом для проверки и читаемости:**
- В `processing.lua` и в соответствующих местах `processing.tex` обязательно оставляй **примеры расчётов**:
  - Для таблиц — первая строка (или первые 2–3 строки) с конкретными значениями и комментариями.
  - Для пошаговых расчётов — промежуточные значения с комментариями.
- Студент/преподаватель должен видеть: «вот исходные данные → вот промежуточный шаг → вот конечный результат» (всё из Lua).

**Оформление заданий:**
    ```latex
    \subsection*{ \boxed{\text{ Задание N. }} }
    \begin{quote}
        \textit{<Текст задания из методички>}
    \end{quote}
    ```

## 4. Python-визуализация и интеграция (lupa)
Скрипты в `/code/*.py` используют расчеты из Lua. Для корректного импорта без конфликтов путей используй следующую структуру инициализации:

    ```python
    import os
    from lupa import LuaRuntime

    # Корректная настройка путей для импорта lua-модулей
    script_dir = os.path.dirname(os.path.abspath(__file__))
    project_root = os.path.dirname(script_dir)
    os.chdir(project_root)

    lua = LuaRuntime(unpack_returned_tuples=True)
    lua.execute("dofile('processing.lua')")
    r = lua.globals().r  # Доступ к глобальной таблице результатов

    # Возврат в директорию скрипта для сохранения графиков
    os.chdir(script_dir)
    # ... логика построения ...
    plt.savefig('plot_name.pgf')
    ```
**Важно:** Сохраняй `.pgf` в текущей директории скрипта. Система верстки сама найдет их в `figs/`.
