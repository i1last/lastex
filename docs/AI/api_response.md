## 1. Технический интерфейс (API)
Используй макросы для связи с расчетным модулем Python:
- `\pv*[формат]{выражение}` — одиночное значение. Поддерживает любой валидный Python-код (например, `\pv*[.2]{R_1 + R_2}`).
- `\pt*[форматы][старт][конец]{массивы}` — генерация строк таблицы. Принимает имена массивов через запятую. `[старт]` начинается с 1. `[конец]` может быть `-1` (до конца). Форматы разделяются запятыми: `[3.1, .2, .2e]`.
- `*` означает, что результат работы будет обернут в `$...$`. Без звездочки просто `...`.

**Форматы (Синтаксис `[a].[b][c]`):**
- `a` — Масштаб (делит на $10^a$). Например, `3` переведет Омы в кОмы.
- `b` — Округление. Количество знаков после запятой.
- `c` — Модификатор. `e` для отображения экспоненциальной формы.
*Примеры:*
- `[.2]`: 1234.5678 -> 1234.58
- `[3.3]`: 1234.5678 -> 1.235
- `[.2e]`: 1234.5678 -> 1.2345678 \cdot 10^{3}
- `[2.2e]`: 1234.5678 -> 12.345678 \cdot 10^{2}

**Обязательное правило для таблиц:**  
Для окружения `tblr` обязательно оборачивать в `\expanded{\pt*[...]{...}}`.

## 2. Архитектура файлов и логика данных
- `protocol.tex`: Таблицы для «сырых» данных. Минимум теории.
- `main.tex`: Цели, задачи, сжатая теория. Подробнейшая инструкция по снятию данных (Что сделать → Как → Нюансы).
- `processing.tex`: Последовательное выполнение заданий. В конце файла **ОБЯЗАТЕЛЬНО** оставить пустой раздел `\section*{Выводы}`.
- `scripts/processing.py`: Логика расчетов. Импортирует `scripts/protocol.py`. **Имя папки строго `scripts`**

**Требования к Python-переменным:**
- Используй максимально короткие, но семантически понятные имена переменных.
- Для организации данных используй словари `p` (протокол) и `r` (результаты) или префиксы (например, `p_U`, `r_R`).
- Все расчетные переменные должны быть доступны в глобальной области видимости модуля для корректного импорта в LaTeX.

## 3. Правила верстки, таблиц и подписей
- **Подписи к элементам:**
  - Таблицы и листинги (`table`, `longtblr`, `listing`) — `\caption{}` **всегда сверху**.
  - Рисунки (`figure`) — `\caption{}` **всегда снизу**.
- **Расширение таблиц:** Для `tblr` и `longtblr` по умолчанию установлен `expand=\expanded`, поэтому `\expanded{}` работает из коробки.
- **Объединение ячеек:** Только `\SetCell[r=rows, c=cols]{align}`.
- **Списки:** Только окружения `itemize` или `enumerate`. Запрет на Markdown-нумерацию.
- **Выделение:** Только `\textit{...}`. Полный запрет на `\textbf{...}`.
- **Ссылки:** Только `\cref{label}`.
- **Математика:** Инлайн `$ ... $`, выносные `equation`/`align`. Запрет на `$$ ... $$`. Числа через точку (`3.14`).

## 4. Требования к обработке в `processing.tex`
**Правило автоматического пересчёта:**
- Все числа берутся ИСКЛЮЧИТЕЛЬНО из Python через макросы `\pv` и `\pt`. Ручной ввод чисел ЗАПРЕЩЕН.
- Это гарантирует, что при изменении данных в `protocol.py` вся лабораторная работа пересчитается автоматически и полностью.
- Инициализация окружения происходит один раз в основном файле: `\begin{python} from scripts.processing import * \end{python}`.

**При этом для проверки и читаемости:**
- В `processing.tex` демонстрируй логику вычислений (1-2 примера для однотипных расчетов).
- Шаблон демонстрации: Символьная формула $\rightarrow$ Подстановка `\pv` $\rightarrow$ Результат `\pv`.
- Пример:
    ```latex
    \begin{equation}
        R = \frac{U}{I} = \frac{\pv*[.2]{p['U']}}{\pv*[.2]{p['I']}} = \pv*[.2]{r['R']} \text{ Ом}
    \end{equation}
    ```
- Студент/преподаватель должен видеть: «вот исходные данные → вот промежуточный шаг → вот конечный результат» (всё из Python).

**Оформление заданий:**
    ```latex
    \subsection*{ \fbox{Задание N.} }
    \begin{quote}
        \textit{<Текст задания из методички>}
    \end{quote}
    ```

## 5. Python-визуализация и графики
⚠️ **Строгое разделение:** Скрипты построения графиков (например, `plot_afc.py`) не должны содержать первичных расчетов. Вся математика находится в `processing.py`. Скрипты графиков запускаются пользователем **вручную и изолированно**.

Выполняй прямой импорт данных из соседних модулей:
    ```python
    import matplotlib.pyplot as plt
    from scripts.processing import r # Импорт готовых результатов расчетов

    # ... логика построения ...
    # Сохранение графика в директории скрипта
    plt.savefig('plot_name.pgf')
    ```
**Важно:**
- СОХРАНЯЙ `.pgf` в ТЕКУЩЕЙ директории скрипта. Система верстки сама найдет их в `figs/`.
- НЕ настраивай стили (размеры шрифтов, стили и цвета линий, расположение легенды, сетка) — они уже настроены в `matplotlibrc`, который применяется автоматически.
- НЕ создавай `title`, его функцию выполняет `\caption{}` в `.tex`.
