# Устранение неполадок

## 1. Проблемы с Docker-контейнером

**Симптом:** Ошибка `command not found`, зависание сборки или сообщение о недоступности демона.
*   **Причина:** Контейнер `latex-daemon` перешел в некорректное состояние или Docker-движок перезапускался.
*   **Решение:**
    1.  Выполнить принудительную остановку: `python lastex.py stop`.
    2.  Запустить сборку заново: `python lastex.py build <путь>`. Система автоматически пересоздаст контейнер.

**Симптом:** Ошибка при генерации образа (Dockerfile).
*   **Решение:**
    1.  Убедитесь, что все проприетарные шрифты (Times New Roman, JetBrains Mono) присутствуют в директории [`core/fonts/`](../core/fonts/).
    2.  Убедитесь в подключении к интернету (старые версии zapret могут блокировать доступ к репозиторию для установки docker образа).

## 2. Ошибки компиляции $\LaTeX$

**Симптом:** Документ не видит новые пакеты, добавленные в `etulab.cls`, или возникают странные ошибки разметки.
*   **Причина:** Рассинхронизация кэша прекомпилированного формата `.fmt`.
*   **Решение:**
    1.  Выполнить жесткую очистку: `python lastex.py clean <путь_к_отчету>`.
    2.  Убедиться, что папка `out/` удалена. При следующем запуске `build` формат будет сгенерирован заново.

**Симптом:** `Package biber Error: Cannot find control file '...bcf'`.
*   **Причина:** Ошибка компиляции на первом проходе, из-за которой не сформировался файл библиографии.
*   **Решение:** Изучите полный лог в `out/название_файла.log`. Исправьте синтаксическую ошибку в тексте и перезапустите сборку.

## 3. Ошибки генерации графики (Python)

**Симптом:** Скрипт выполняется, но график не обновляется в PDF.
*   **Причина:** Файл `.pgf` не был создан или скрипт завершился с ошибкой (Traceback).
*   **Решение:**
    1.  Запустите изолированную сборку скрипта: `python lastex.py build путь_к_скрипту.py` или `Ctrl + Alt + B`, открыв и зафиксировав курсор на вкладке с файлом.
    2.  Проверьте вывод консоли на наличие исключений Python.
    3.  Убедитесь, что в коде присутствует команда сохранения файла: `plt.savefig('filename.pgf')`. Скрипт-обертка ищет именно файлы с расширением `.pgf` для их перемещения в `figs/`.
